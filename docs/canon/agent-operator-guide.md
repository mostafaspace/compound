# Compound Agent Operator Guide

Status: Canonical
Last updated: 2026-04-29

## Startup Rules

1. Read `docs/canon/README.md`.
2. Read the canon sections relevant to the task.
3. Treat Jira as execution tracking only.
4. Define the impact area before touching code.
5. Do not trust legacy docs over canon.

## Working Rules

- Start from canon plus the current task scope.
- Use Jira to understand what is being executed, blocked, or tested.
- Do not let a Jira story redefine older platform behavior unless canon has been updated first.
- Record validation evidence, not just conclusions.

## Handoff Rules

- Leave the repo in a traceable state.
- Update Jira with evidence if Jira is part of the slice.
- If important context exists only in chat, write it into canon, audit notes, or Jira evidence before stopping.
